Este proyecto está diseñado para permitir una escalabilidad completa hacia la arquitectura DDD. 
Al separar la lógica de negocio y los datos, se facilita la introducción de nuevas funcionalidades y 
la adaptación a cambios sin comprometer la estructura existente. Esta organización modular asegura que el 
proyecto pueda evolucionar y manejar una mayor complejidad a medida que se desarrollen nuevas características
en el futuro. Además, se utiliza Swagger para mantener una documentación clara y accesible, lo que facilita 
otros desarrolladores comprender y utilizar el código.


INSTRUCCIONES DE USO:

- Proyecto de Gestión de Tareas Este proyecto consiste en una API REST para la gestión de tareas y una aplicación frontend en React que consume esta API.
A continuación se presentan las instrucciones para ejecutar ambos componentes. 
* Requisitos Previos Asegúrate de tener instalados los siguientes programas en tu máquina: - [Node.js](https://nodejs.org/) (v12 o superior)
 - Instrucciones para Ejecutar la API 
1. Navega a la carpeta de la API: cd trf 
 2. Instala las dependencias: npm install 
 3. Ejecuta la API:  npm start  La API estará disponible en `http://localhost:3000`.
 - Instrucciones para Ejecutar la Aplicación de React 
1. Navega a la carpeta del frontend: ``` cd front ```
 2. Instala las dependencias:  npm install
 3. Ejecuta la aplicación:  npm start  La aplicación estará disponible en `http://localhost:3001`.
* Uso Una vez que tanto la API como la aplicación frontend estén en funcionamiento, puedes acceder a la aplicación en tu navegador. Podrás agregar, eliminar y marcar tareas como completadas.
